プロジェクト名: 誰もが自炊ができるようになる自炊支援基盤とVRシステム

申請者名: 奥村英貴、浦川樹

# なにをつくるか

　料理は日々の生活に欠かせない活動である。しかし、少なからず苦手とされる技能の一つでもある。その理由として、手際が悪かったりレシピ外の文脈を読み違えて調理に失敗してしまうことや、狭いキッチンで自分の中でレシピ工程の実行を計画して行うのがめんどくさいことなどが考えられる。こうした課題感に対して、多くの先行研究がなされてきた。調理工程をグラフの形で表現することで、ユーザーが料理に必要な道具や材料を把握し、調理全体を俯瞰できるようにするシステム[1]や、工程の実行計画を道具や場所のリソース内で計画しユーザーは工程の実行のみに集中できるようにするシステム[2]、調理道具にセンサーを付けることで調理工程の進捗を管理するシステム[3]などがある。しかし、現状こうしたシステムは普及しておらず、従来の箇条書き形式のレシピサイトが支配的である。料理の失敗や面倒さ、再現性を解消するシステムは少なくとも私自身がその存在を求めていることから、需要が0というわけではない。そこで私たちは、成功しているナビゲーションシステムであるGoogleMapに習うことに加え、道路案内にはない料理固有のドメインを見つめ直すことで、「レシピを選択し、実施準備が整ったら、あとは指示に従うだけで料理ができる」というユースケースを一般のユーザーに向けて実現する調理ナビゲーションシステムを作成する。さらにこの調理ナビゲーションシステムを基盤とし、コンピュータによる支援をより直感的にするため、VRシステムとの統合を目指す。

GoogleMapが先行研究[1][2][3]で提案されたシステムより優れている点は以下の3つである。

1. ナビゲーション画面には局所的な必要性に基づいて情報が配置されている
2. 工程が進む瞬間を直感的に把握することができる
3. 工程の内容がユーザーにとって直感的である
4. 目的対象の選択から実施、完了までをすべてサポートしている

これらの観点を考慮することで、直感的に調理をナビゲーションできる実用的なシステムを構築することができる。

ここで留意すべきは、料理というドメインにはGoogleMapでは扱っていない複雑さが存在するという事実である。調理は以下のように三つの要素に分解される。

**料理**
- 調理準備: 調理実施のための素材や道具の調達
- 調理技術: 調理工程を正確に効率よく実施するために学習しなければならない要素技術とユーザーごとの習熟度
- 調理計画: 調理工程をどのような順番で、どの道具を使って実施するかという調理実行計画

ここで調理準備については物流の進化とともに調理実施の障壁には比較的なっていないことから、調理技術と調理計画について注目する。GoogleMapが実施する経路ナビゲーションと比較して、料理ナビゲーションは以下のような複雑さを有している。

**調理技術**
1. 多種多様な要素技術が存在する (Ex. 「焼き」「蒸し」「揚げ」)
2. 要素技術についての習熟度合いは人それぞれである (Ex. 「焼きはたくさん経験があるが、蒸しはほとんどやったことがない」)

**調理計画**
1. ほとんどの工程が後戻りできない (Ex. 「銀杏切りした人参を乱切りした人参にすることはできない」)
2. 並列実行を前提としており、割り込みが発生する (Ex. 「スープをときどきかき混ぜながら1時間煮る、その間にゆずをみじん切りにする」)
3. 工程に時間的制約があり、手際よくやる必要がある (Ex. 「麺が茹で上がるまでに、ソースを完成させておく。茹で上がりと同時に完成するのが理想」)
4. ふとした操作が結果に大きな影響を及ぼす (Ex. 「ふと蓋をしたらパリパリにならなかった」「強くこしすぎて濁りが入った」)
5. 工程終了が対象の状態変化によって規定される工程の存在 (Ex. 「鶏ももをこんがりするまで焼く」)
6. 状態の変化が匂いや味といった画像/動画/テキスト/音声では表現しづらい要素をもとにしている (Ex. 「にんにくの香りが立ったら火を止める」)

もちろんレシピによって複雑さの程度はさまざまであり、GoogleMap的な案内だけで十分実施可能なレシピも少なからずある。しかし、多くの料理を安全にナビゲーションするためにはこれらの複雑さを取り込んだアプリケーションを作成する必要がある。

以上を考慮し、本プロジェクトでは以下の要件

1. 要素技術それぞれに対して免許皆伝要素を導入する -> ユーザーの習熟技術に合わせたナビゲーションが可能に
2. 割り込みが発生しそうな状況を直感的に伝えるUI -> 多並列な状況でもあたふたしないように
3. 工程の内容を機械命令のように明確化 -> ユーザーの工程習熟を促す
4. ユーザーの手際に合わせたレシピ実施難度の計算 -> 現在の技術種熟度では困難な調理を実施前に知ることが可能に
5. 機械学習やユーザー知覚アンケートによる工程終了条件の明確化 -> 初心者でも安心して工程を進められるように
6. レシピで管理しきれない詳細部での落とし穴を公募 -> 先人に学ぶことが可能に
7. 先行研究のシステムのいいところ(Ex. レシピグラフ化、技能と設備を把握し実施計画を設計)を取り込む -> 料理でナビゲーションを実現、多料理でも対応可能

を実装することで、各ユーザーの技量に合わせて目的の料理まで調理をナビゲートしてくれるシステムを構築する。

具体的には以下の流れで実装する。

1. 技能、レシピといったドメイン設計
2. レシピ作成機能
3. レシピ実行計画システム
4. レシピナビゲーションシステム
5. レシピFB機能
6. レシピ検索システム

本プロジェクトの段階的な目標を調理技術と調理計画のそれぞれに対して示す。

**調理計画**
1. 料理固有の複雑さが少ない料理(Ex. 炒め物)に対してナビゲーションができる
2. 複雑さが少ない料理を複数、同時並行でナビゲーションできる
3. 複雑さが伴う料理(Ex. 焼き、煮物、etc..)に対してナビゲーションできる
4. 複雑さが伴う料理でも複数料理、同時並行でナビゲーションできる

**調理技術**
1. 要素技術を固定数に絞り、それぞれについて習熟度を認定する方法を設計する
2. 要素技術を拡大できるように拡張する

さらに最終的な目標として、ナビゲーションシステムからの指令とユーザーからの工程完了の合図をよりシームレスに伝達し合うためのインターフェースとしてVRアプリケーションの構築を計画している。最初は、工程完了の判定をシステムの側から打診することは難しいため、ユーザーからのジェスチャ合図などで検知することになる。しかし最終的には、VRシステムで取得できる工程実施ビデオと工程を表すテキストの対応が取れたデータを多数取得できるため、それらを用いて、GoogleMapのように、システムの側から次の工程を提示していくこともできるようになる。


# どんな出し方を考えているか

できるだけWebで完結するように実装し、Webサービスとして公開する。計算時間や機器接続などの問題でWebでの完結が困難な場合は、ユーザーの利用方法を考慮して、ナビゲーションシステム部などをネイティブアプリに切り出す。本システムを用いて自炊をするユーザー層にボリュームがでれば、ビジネス展開に持ちこむことを狙う。具体的は、調理器具メーカーや食品メーカーなどに対して、彼らの商品の利便性をアピールできる窓口として有料で使ってもらうなどを考えている。

# 斬新さの主張、期待される効果など

> 少し背伸びをしてもいいので、自由に記述してください。

本システムは以下の点で斬新である。

1. ユーザーの技量に合わせた柔軟な対応
2. 大量の行動データとテキスト指令の対応から料理を革新
3. 世界へのビジネス展開の可能性

以下それぞれ説明する。

## ユーザーの技量に合わせた柔軟な対応

現状のレシピサイトは、自分で調理計画を立てられ、調理技術も事前に備えているユーザー層にとっては十分な内容である。しかし、その両方に慣れているわけではない層には不親切である。先行研究で示されたシステムは、調理計画について良い示唆を与えるが、調理技術についてのサポートがなく、インターフェースも煩雑で十分実用的であるとはいえない。こうした現状のもとで本システムでは、ユーザー固有の調理技術に対して調理計画の実施困難さが決まるという当たり前の性質に目をつけ、さらにそれをコンピュータ上で計算できる形に落としている。(Ex. 各ユーザーごとに工程実施にかかる時間を計測し、その時間効率で計画実施成功の可能性を計算する)こうした工夫によって、ユーザーの習熟度や希望に合わせて、柔軟にレシピ実行計画やナビゲーションを調整できるようになっている。それにより、従来のシステムでは取り残されていたユーザー層が料理界隈に参加できるようになっていくことが期待される。

## 大量の行動データとテキスト指令の対応から料理を革新

本は、単なる自炊支援ツールにとどまらず、人間の行動データとテキストによる指令の対応という貴重なデータセットを大量に生成することができる。このデータセットは、料理の自動化やAIアシスタントの開発など、さまざまな分野における技術革新に役立てられる可能性を秘めている。

具体的には、以下の効果が期待できる。

- 料理の自動化：ユーザーの行動データから、調理の各工程を自動化するアルゴリズムを開発することが可能になる。
- AIアシスタントの開発：テキスト指令と行動データの対応関係を分析することで、より自然な言語で指示を受け付けるAIアシスタントを開発することが可能になる。
- 料理の学習・教育：VR空間で料理を体験することで、より効果的に料理を学ぶことができる。
- レシピの開発・改善：ユーザーの行動データから、より効率的で美味しいレシピを開発することができる可能性がある。

本システムは、料理を革新し、食生活を改善するだけでなく、AI技術の発展にも大きく貢献する可能性を秘めているのだ。

## 世界へのビジネス展開の可能性

自炊は日本だけでなく、世界中で行われている活動である。そのため、言語や地域性にとらわれない本システムは、世界中の自炊をするユーザー層にサービスを拡大できる可能性を秘めている。もし世界的にユーザー層が拡大すれば、調理器具メーカーや食品メーカーにとってより魅力的な広告媒体になる。さらに、単純に世界の料理に対応していくだけでなく、現在の音楽シーンがそうであるように、それぞれの料理のノウハウや哲学が混ざり合うことで、新しいジャンルの美味しい料理が生み出されるようになることが期待される。
